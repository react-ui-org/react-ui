# Theming

import { Playground } from 'docz'

From the very beginning, React UI has been designed with a great emphasis on
customizability. We decided to leverage CSS custom properties for this feature
for two main reasons:

1. We believe in **power of native CSS**. Preprocessors are still a thing, but
   it's not necessary to go as far as for CSS-in-JS to make a UI customizable.

2. Thanks to its JavaScript API, CSS custom properties are both **readable and
   writable from JS code**.

## Theming Options

CSS custom properties are used to define common visual properties like colors,
fonts, borders, shadows, or spacing. They come prefixed with `rui-` so they
don't get in way of other custom properties in your project.

Example `theme.scss`:

```scss
:root {
  // Brand colors
  --rui-color-primary: #00778b;
  --rui-color-primary-light: #{lighten(#00778b, 70%)};
  --rui-color-primary-dark: #{darken(#00778b, 4%)};
  --rui-color-primary-darker: #{darken(#00778b, 8%)};
  --rui-color-on-primary: #fff;
  --rui-color-secondary: #fa4616;
  --rui-color-secondary-light: #{lighten(#fa4616, 45%)};
  --rui-color-secondary-dark: #{darken(#fa4616, 4%)};
  --rui-color-secondary-darker: #{darken(#fa4616, 8%)};
  --rui-color-on-secondary: #fff;
}
```

It is also possible to adjust some properties on individual components level,
preferably by reusing the global settings:

```scss
:root {
  // Alerts: common properties
  --rui-alert-border-width: var(--rui-border-width);
  --rui-alert-border-radius: var(--rui-border-radius);
  --rui-alert-padding: var(--rui-spacing-2);
}
```

You can adjust any of these options in your styles. See the
[default theme](https://github.com/react-ui-org/react-ui/blob/master/src/lib/theme.scss)
for the full list of available settings.

## Forms

### Basic Theming

The following theme options define basic appearance of form fields. The options
can be divided into three groups by their specificity:

#### Common Options

Theming options affecting all form components.

| Custom Property                                      | Description                                                  |
|------------------------------------------------------|--------------------------------------------------------------|
| `--rui-form-field-label-color`                       | Label text color                                             |
| `--rui-form-field-label-font-size`                   | Label font size                                              |
| `--rui-form-field-help-text-font-size`               | Help text font size                                          |
| `--rui-form-field-help-text-font-style`              | Help text font style, eg. italic                             |
| `--rui-form-field-help-text-color`                   | Help text color                                              |
| `--rui-form-field-required-sign`                     | Text appended to required input labels                       |
| `--rui-form-field-required-sign-color`               | Color of text appended to required input labels              |
| `--rui-form-field-disabled-cursor`                   | Cursor to show on hovering disabled form fields              |
| `--rui-form-field-disabled-opacity`                  | Opacity of disabled form fields (inc. label and help text)   |
| `--rui-form-field-horizontal-label-alignment`        | Text alignment of labels in horizontal layout                |
| `--rui-form-field-horizontal-label-min-width`        | Minimum width of labels in horizontal layout                 |
| `--rui-form-field-horizontal-label-width`            | Default width of labels in horizontal layout                 |
| `--rui-form-field-horizontal-full-width-label-width` | Default width of labels in full-width horizontal layout      |

#### Shared TextField, TextArea and SelectField Options

Options shared by some components. This includes
[TextField](/components/ui/text-field),
[TextArea](/components/ui/text-area), and
[SelectField](/components/ui/select-field).

| Custom Property                                      | Description                                                  |
|------------------------------------------------------|--------------------------------------------------------------|
| `--rui-form-field-input-width`                       | Default text input and select box width                      |
| `--rui-form-field-input-min-width`                   | Minimum text input and select box width                      |
| `--rui-form-field-border-width`                      | Control border width                                         |
| `--rui-form-field-border-radius`                     | Control corner radius                                        |
| `--rui-form-field-placeholder-color`                 | Placeholder text color                                       |

#### SelectField Specific Options

| Custom Property                                      | Description                                                  |
|------------------------------------------------------|--------------------------------------------------------------|
| `--rui-form-field-caret-border-style`                | SelectField arrow border style (eg. solid)                   |
| `--rui-form-field-caret-background`                  | SelectField arrow background                                 |
| `--rui-form-field-disabled-select-option-color`      | Text color of disabled SelectField options                   |

#### Validation States

Theming options for validation states are shared by all components. Naming
convention looks as follows:

`--rui-form-field-<VALIDATION STATE>-<INTERACTION STATE>-<PROPERTY>`

Where:

- `<VALIDATION STATE>` is one of `invalid`, `valid`, or `warning`
- `<INTERACTION STATE>` is one of `default` or `disabled`
- `<PROPERTY>` is one of `color`, `border-color`, `background`, `box-shadow`, or
  `surrounding-text-color`

Not all properties are used by all components, this varies from component to
component. Also, `hover` interaction state is unavailable for validation states,
`default` state appearance is retained on hovering. `focus` state is inherited
from default form field appearance (i.e. as if no validation state was set).

Example:

<Playground>
{() => {
  const [fruit, setFruit] = React.useState('apple');
  const options = [
    {
      label: 'Apple',
      value: 'apple',
    },
    {
      label: 'Banana',
      value: 'banana',
    },
    {
      label: 'Grapefruit',
      value: 'grapefruit',
    },
  ];
  return (
    <>
      <style type="text/css">
        {`
        .example--themed-form-field-validation-states {
          --rui-form-field-valid-default-color: white;
          --rui-form-field-valid-default-background: var(--rui-color-success);
        }
      `}
      </style>
      <TextField
        id="default-valid-text-field"
        label="Default valid text field"
        validationState="valid"
        validationText="This field is valid"
      />
      <SelectField
        changeHandler={(e) => setFruit(e.target.value)}
        id="default-valid-select-field"
        label="Default valid select field"
        options={options}

        validationText="This field is valid"
        value={fruit}
      />
      <div className="example example--themed-form-field-validation-states">
        <TextField
          id="themed-valid-text-field"
          label="Themed valid text field"
          validationState="valid"
          validationText="This field is valid"
        />
        <SelectField
          changeHandler={(e) => setFruit(e.target.value)}
          id="themed-valid-select-field"
          label="Themed valid select field"
          options={options}
          validationState="valid"
          validationText="This field is valid"
          value={fruit}
        />
      </div>
    </>
  )
}}

#### Text Control and Select Box Variants

Theming options for textual controls and select boxes:
[TextField](/components/ui/text-field),
[TextArea](/components/ui/text-area), and
[SelectField](/components/ui/select-field).
Naming is very similar to validation state options:

`--rui-form-field-<VISUAL VARIANT>-<INTERACTION STATE>-<PROPERTY>`

Where:

- `<VISUAL VARIANT>` is one of `filled` or `outline`
- `<INTERACTION STATE>` is one of `default`, `hover`, `focus`, or `disabled`
- `<PROPERTY>` is one of `color`, `border-color`, `background`, `box-shadow`, or
  `surrounding-text-color` (the last one being available only for
  `default` and `disabled` interaction states)

### Form Field Sizes

Available sizes can be adjusted as follows:

`--rui-form-field-<SIZE>-<PROPERTY>`

Where:

- `<SIZE>` is one of `small`, `medium`, or `large`
- `<PROPERTY>` is one of `height`, `padding-x`, `padding-y`, or `font-size`

### Form Layout

| Custom Property                                      | Description                                                  |
|------------------------------------------------------|--------------------------------------------------------------|
| `--rui-form-layout-horizontal-label-auto-width`      | Width of labels in horizontal form layout, `auto` sizing     |
| `--rui-form-layout-horizontal-label-limited-width`   | Width of labels in horizontal form layout, `limited` sizing  |
| `--rui-form-layout-horizontal-label-default-width`   | Width of labels in horizontal form layout, `default` sizing  |

## Best Practices

It's a good idea to start with changing the **global settings first**. Widely
reused settings such as colors, typography, borders or spacing values should be
adjusted first because they define the basic appearance of all components.

Having finished the customization at the global level, you can **then proceed to
customize the appearance of individual components** ‚Äî if necessary at all. Even
then you should also reuse existing global settings as much as possible to
ensure that your UI is consistent and works as a system.

For the same reason, if you have any custom components in your UI, you should
**reuse the global theming options in your own CSS**, too.
